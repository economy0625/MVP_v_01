# Product — 제품 정의 및 기획

## 목적
테크잇다AI가 무엇을 만드는지, 누구를 위한 제품인지 정의한다.
모든 기능 개발의 기준점이 되는 문서다.

---

## 제품 비전

"제조기업이 정부지원사업을 남의 도움 없이 스스로 발굴하고, 합격할 수 있게 한다"

---

## 핵심 고객 (Primary User)

**대구·경북 제조 중소기업 대표 / R&D 담당자**
- 지원사업은 알고 싶지만 어디서 찾아야 할지 모름
- 사업계획서를 처음 써보거나 매번 외주에 의존
- KPI를 어떻게 설정해야 하는지 기준이 없음
- 합격 가능성을 객관적으로 알고 싶음

---

## MVP 핵심 사용자 시나리오

```
1. 기업 정보 입력 (5분)
   → 업종, 단계, TRL, 지역, 보유 인증

2. 맞춤 지원사업 추천 확인 (1분)
   → 상위 3개 추천 + 마감일 알림 설정

3. 선정 확률 확인 (1분)
   → 점수 + 부족 항목 피드백

4. KPI 자동 생성 + 물리 검증 (2분)
   → 정량 목표 생성 + 달성 가능성 검토

5. 사업계획서 초안 다운로드 (1분)
   → 즉시 편집 가능한 초안 제공

총 소요 시간: 10분 이내
```

---

## MVP 범위 경계

| 포함 (Phase 1) | 제외 (Phase 2~) |
|---------------|----------------|
| 기업 정보 입력 | MES/ERP 연동 |
| 지원사업 추천 | 자동 크롤링 |
| 선정 확률 예측 | ML 모델 고도화 |
| KPI 생성 + 물리검증 | R&D PMS |
| 계획서 초안 생성 | 정산 증빙 자동화 |
| 전문가 기본 매칭 | 컨소시엄 게시판 |

---

## Claude 작업 규칙

- 새 기능 요청 시 MVP 범위 여부를 이 문서 기준으로 판단
- 사용자 시나리오 변경 시 user_flow.md 업데이트
- 목표 고객 정의 변경 시 target_customer.md 수정
- "좋은 기능"이어도 MVP 범위 밖이면 추가 금지
