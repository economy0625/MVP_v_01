# Scoring Domain — 선정 확률 예측 엔진

## 목적
기업 입력 데이터와 사업계획서 분석을 통해 지원사업 합격 가능성을 점수로 산출하고,
부족 항목에 대한 피드백을 제공한다.

---

## 평가 구조 (5개 항목, 각 20점)

| 항목 | 배점 | 평가 기준 |
|------|------|----------|
| 문제 정의 | 20점 | 시장 문제의 명확성, 구체성 |
| 기술성 | 20점 | TRL 수준, 차별성, 특허 여부 |
| 사업성 | 20점 | 시장 규모, 수익 모델, 경쟁사 분석 |
| 수행역량 | 20점 | 팀 경력, R&D 이력, 협력기관 |
| 계획 타당성 | 20점 | KPI 존재, 일정 합리성, 예산 적정성 |

총점: 100점

---

## 점수 계산 원칙

1. 입력 데이터 기반 계산 (정성 → 정량 변환 필수)
2. 각 항목별 세부 체크리스트로 점수 산정
3. 감점 요소 반드시 반영
4. 최종 확률은 sigmoid 함수로 변환

---

## 감점 요소

| 감점 조건 | 감점 |
|----------|------|
| KPI 없음 | -15점 |
| 협력기관 없음 | -10점 |
| R&D 수행 이력 없음 | -8점 |
| 특허/인증 없음 | -5점 |
| 시장 규모 데이터 없음 | -7점 |

---

## 확률 변환 공식

```
raw_score = 각 항목 합산 (0~100)
adjusted_score = raw_score - 감점 합계
probability = sigmoid(adjusted_score) × 100%

sigmoid(x) = 1 / (1 + e^(-(x - 50) / 15))
```

---

## 출력 형식

```json
{
  "total_score": 74,
  "probability": 68.3,
  "breakdown": {
    "problem": 16,
    "technology": 18,
    "business": 14,
    "capability": 12,
    "plan": 14
  },
  "deductions": ["KPI 없음 (-15점)"],
  "feedback": ["KPI를 정량 목표로 구체화하세요", "협력기관을 추가하면 +10점"]
}
```

---

## Claude 작업 규칙

- 새로운 점수 항목 추가 시 반드시 이 문서 업데이트
- 점수 기준 변경 시 전체 항목 일관성(합계 100점) 유지
- 확률 계산 로직은 항상 explainable하게 유지
- 감점 항목 변경 시 scoring_model.md도 동시 수정
