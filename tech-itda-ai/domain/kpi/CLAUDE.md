# KPI Domain — KPI 자동 생성 + 물리 검증 엔진 (특허 핵심)

## 목적
공정/사업 목표 기반으로 정량 KPI와 기술 설명을 자동 생성하고,
제조 물리 수식으로 KPI 실현 가능성을 검토·보정한다.
→ 플랫폼의 정체성이자 타사가 흉내 낼 수 없는 독보적 IP.

---

## 핵심 기능 2가지

### 1. KPI 자동 생성
- 입력: 사업 목표, 업종, 기술 분야
- 출력: 정량 KPI + 검증 방법 + 달성 기간

### 2. 물리 모델 검증 (차별화 핵심)
- 제조 물리 수식 기반으로 KPI 달성 가능성 계산
- 불가능한 KPI는 보정값 제안
- 전문가 검수와 연계 가능

---

## KPI 생성 구조

### 입력 항목
| 항목 | 예시 |
|------|------|
| 기술 목표 | 경량화 소재 개발 |
| 업종 | 자동차 부품 제조 |
| 목표 수치 (있을 경우) | 중량 30% 감소 |
| 개발 기간 | 12개월 |

### KPI 출력 템플릿
```
목표: [기술 목표]
KPI: [정량 수치] [단위] [달성 기간]
검증 방법: [KS/ISO 규격 or 자체 측정 방법]
물리 검증 결과: 달성 가능 / 보정 필요
보정 제안: (물리적으로 불가능한 경우만)
```

---

## 물리 검증 로직

### 적용 물리 모델 예시
| 분야 | 물리 수식 | KPI 적용 예 |
|------|----------|------------|
| 경량화 | 밀도 × 체적 = 질량 | 중량 감소율 한계 계산 |
| 강도 | σ = F/A | 인장강도 KPI 타당성 |
| 열처리 | 아레니우스 방정식 | 경도 향상 가능 범위 |
| 에너지효율 | η = W_out/W_in | 효율 개선 상한 계산 |

### 검증 흐름
```
KPI 입력 → 해당 물리 모델 선택 → 수식 계산 → 달성 가능성 판정
                                              ↓
                                  가능: 승인 출력
                                  불가: 보정값 제안
```

---

## 출력 예시

```json
{
  "kpi": "인장강도 350MPa 이상 달성 (12개월 내)",
  "verification_method": "KS B 0802 인장시험",
  "physics_check": {
    "status": "달성 가능",
    "basis": "현재 소재 기준 최대 380MPa 물리적 상한 내",
    "formula": "σ = F/A, 안전율 1.2 적용"
  }
}
```

## Claude 작업 규칙

- KPI 템플릿 추가 시 반드시 kpi_templates.md 업데이트
- 물리 수식 모델 추가 시 kpi_generation_logic.md에 추가
- 물리 검증 결과는 항상 근거 수식과 함께 출력
- 전문가 검수 플래그(needs_expert_review)는 복잡한 경우 true로 설정
- 특허 관련 로직은 외부 노출 금지
