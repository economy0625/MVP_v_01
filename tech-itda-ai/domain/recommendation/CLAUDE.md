# Recommendation Domain — 지원사업 추천 엔진

## 목적
기업 역량 스코어(입력 데이터)를 기반으로 최적의 정부지원사업을 추천하고,
신청 타이밍 알림을 제공하여 사용자 재방문(Stickiness)을 유도한다.

---

## 핵심 입력값 (기업 데이터)

| 항목 | 설명 | 예시 |
|------|------|------|
| industry | 업종 코드 | 제조업, 소재부품 |
| company_size | 기업 규모 | 예비창업, 초기, 성장 |
| tech_level | 기술 수준 | TRL 1~9 |
| revenue | 매출 규모 | 0, 5억, 50억 |
| cert | 보유 인증 | ISO, 이노비즈, 벤처 |
| location | 지역 | 대구, 서울 |
| prior_rd | R&D 수행 이력 | 없음 / 1건 / 3건 이상 |

---

## 추천 로직 (Rule 기반)

### Rule 1: 기업 단계 매칭
- 예비창업 → 예비창업패키지, TIPS 예비
- 초기(3년 이하) → 초기창업패키지, 디딤돌
- 성장기 → 스케일업, 소부장, 산업부 R&D

### Rule 2: 기술 수준(TRL) 매칭
- TRL 1~3 → 기초연구, 기획과제
- TRL 4~6 → 개발과제 (중기부, 산업부)
- TRL 7~9 → 실증, 사업화 과제

### Rule 3: 지역 우선 가중치
- 지역 특화 사업 존재 시 +10점 가산
- 예: 대구 → 대구테크노파크, 경북 강소기업 등

### Rule 4: 인증 보유 가산점
- 벤처: +5점
- 이노비즈: +7점
- ISO: +3점

## 추천 결과 출력 형식

```json
{
  "rank": 1,
  "program_name": "예비창업패키지",
  "match_score": 87,
  "reason": ["기업 단계 일치", "TRL 매칭", "지역 우선"],
  "deadline": "2025-03-31",
  "alert_timing": "D-30, D-7"
}
```

## Claude 작업 규칙

- 추천 Rule 변경 시 반드시 rules.md 업데이트
- 새로운 사업 유형 추가 시 매칭 로직 전체 검토
- 점수 계산은 scoring 도메인과 독립적으로 유지
- 추천 이유(reason)는 항상 한국어로 explainable하게 출력
