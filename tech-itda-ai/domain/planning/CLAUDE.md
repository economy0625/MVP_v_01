# Planning Domain — 사업계획서 초안 생성 엔진

## 목적
공고별 평가 지표에 최적화된 맞춤형 R&D 사업계획서 초안을 LLM 기반으로 자동 생성한다.
사용자가 입력한 기업 정보 + KPI + 추천 사업을 조합하여 즉시 사용 가능한 초안을 제공한다.

---

## 입력 → 출력 구조

### 필수 입력
| 항목 | 출처 |
|------|------|
| 기업 기본 정보 | data/ (기업 스키마) |
| 지원사업 공고문 | recommendation 도메인 |
| 생성된 KPI | kpi 도메인 |
| 선정 확률 피드백 | scoring 도메인 |

### 출력 (계획서 섹션)
1. 사업 개요 및 필요성
2. 기술 개발 목표 (KPI 자동 삽입)
3. 개발 내용 및 방법
4. 추진 일정
5. 기대 성과 및 활용 방안
6. 보유 역량 및 수행 체계

---

## 프롬프트 설계 원칙

1. 공고별 평가 항목을 섹션 헤더로 구조화
2. KPI는 물리 검증 완료된 값만 사용
3. 감점 요소(scoring 도메인)를 역으로 커버하는 문장 생성
4. 정성적 표현 → 정량 수치로 최대한 변환
5. 출력 길이: 섹션당 200~400자 (초안 기준)

---

## 섹션별 프롬프트 템플릿

### 사업 개요
```
[업종]의 [핵심 문제]를 해결하기 위해 [기술 접근법]을 개발한다.
현재 [문제 현황 수치]이며, 본 과제를 통해 [KPI]를 달성한다.
```

### 기술 개발 목표
```
본 과제의 정량적 목표는 다음과 같다:
- KPI 1: [값] [단위] ([검증 방법])
- KPI 2: [값] [단위] ([검증 방법])
```

### 기대 성과
```
기술적 성과: [KPI 달성]
경제적 성과: [매출 목표, 고용 창출]
파급 효과: [산업 적용 범위]
```

---

## 정책/규정 자동 반영

- 창업 규정, 과제 운영 규정을 실시간 반영
- 위반 리스크 항목 자동 감지 및 경고
- 예: "매출액 기준 기업 규모 초과 시 참여 불가" 자동 체크

---

## Claude 작업 규칙

- 계획서 프롬프트 수정 시 prompt_design.md 반드시 업데이트
- 새로운 공고 유형 추가 시 output_templates.md에 템플릿 추가
- KPI 값은 kpi 도메인 출력값을 그대로 참조 (임의 생성 금지)
- 모든 수치는 근거 있는 값만 사용 (hallucination 방지)
- 생성된 초안은 반드시 "초안" 레이블 표시
